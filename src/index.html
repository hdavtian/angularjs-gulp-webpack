A simple website package v2..

This is a simple website package powered by gulp ... will help you setup a dev site and develop faster!

Once setup and running, the gulp watch command will launch a new browser window, and your html, sass and js files will be watched and upon any changes the browser will auto reload.

Current Features!

  • sass
  • browser-sync
  • and more ...

How do the .js files work

There are two sub-directories in the src/js folder, 1) src/js/site, 2) src/js/vendor. The idea is that you put all your custom js files in #1 and all vendor/3rd party libraries in #2. The gulp task will watch for changes in these folders and on each save will create a concatenate and minified file for each folder and copy to the build folder. If you need a particular .js file to load before another, then specify that setting in gulpfile.js for the approrpriate task (either scripts-site or scripts-vendor)

So for example you may start with:

    [src]
        [js]
            [site]
                one.js
                two.js
                three.js
            [vendor]
                jquery.js
                bootstrap.js
                angular.js

On each save, the grunt task will create the following:

    [build]
        [js]
            site.min.js
            vendor.min.js

Your markup files will reference these two minified files just before the closing body tag.
